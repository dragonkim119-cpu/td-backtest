## 1. 프로젝트 개요

**DeMark TD Sequential** 인디케이터를 비트코인 선물(Binance BTCUSDT) 차트에 적용해 다음을 수행하는 개인용 웹 대시보드:

- 과거 데이터에서 TD 시그널(Setup 9, Countdown 13 등) 자동 검출
- 시그널 발생 위치를 차트에 시각화
- 시그널 이후 N봉 수익률 통계 산출
- 멀티 심볼·멀티 타임프레임 스크리너
- Setup 9 진입 / Risk Level 손절 기반 PnL 백테스트
- 신규 시그널 발생 시 실시간 텔레그램 알림

## 5. TD Sequential 개념

TD Sequential은 Tom DeMark가 개발한 가격 소진(exhaustion) 기반 반전 인디케이터. 추세의 끝을 수량화해 잠재적 반전 시점을 포착.

### 5.1 Setup Phase (1~9)

**조건**: `종가[i] < 종가[i-4]` (매수) 또는 `종가[i] > 종가[i-4]` (매도) 가 **9봉 연속** 성립.

```
매수 Setup: 현재 봉 종가 < 4봉 전 종가 (9번 연속)
매도 Setup: 현재 봉 종가 > 4봉 전 종가 (9번 연속)
```

- 9번째 봉에서 Setup 완성 — 추세 소진 신호
- 중간에 조건이 깨지면 카운트 0 리셋

### 5.2 Perfected Setup

Setup 9 완성 시 추가 조건 충족 여부 확인:

| | 조건 |
|---|---|
| 매수 Perfected | 8번·9번째 봉의 저가 중 하나 ≤ 6번·7번째 봉의 저가 |
| 매도 Perfected | 8번·9번째 봉의 고가 중 하나 ≥ 6번·7번째 봉의 고가 |

**Perfected Setup은 더 강한 반전 신호**. 차트에서 `9✓` 로 표시됨.

### 5.3 TDST (TD Setup Trend)

Setup 완성 시 생성되는 핵심 지지/저항선:

| | 계산 |
|---|---|
| TDST 지지선 (매수 Setup 완성 시) | Setup 9봉 구간의 최저 True Low |
| TDST 저항선 (매도 Setup 완성 시) | Setup 9봉 구간의 최고 True High |

- True High = max(고가, 직전봉 종가)
- True Low = min(저가, 직전봉 종가)

**의미**: 이 선 아래(지지선 기준)로 종가가 마감되면 Countdown이 취소됨.

### 5.4 Countdown Phase (1~13)

Setup 완성 후 반전 신호를 구체화하는 비연속 카운트:

```
매수 Countdown: 종가[i] ≤ 저가[i-2]  조건 만족 시마다 +1 (비연속)
매도 Countdown: 종가[i] ≥ 고가[i-2]  조건 만족 시마다 +1 (비연속)
```

- 조건 불만족 봉에서도 카운트는 유지됨 (리셋 없음)
- 13봉째에 Countdown 완성 — 강한 반전 신호

### 5.5 13 vs 8 Deferral (보류)

Countdown 13번째 봉이 다음 조건을 미충족하면 보류(+):

| | 조건 |
|---|---|
| 매수 | 13번째 봉의 저가 ≤ 8번째 봉의 종가 |
| 매도 | 13번째 봉의 고가 ≥ 8번째 봉의 종가 |

보류 상태에서 이후 봉이 조건을 충족하면 그때 13 완성. 차트에서 `13+` 로 표시됨.

### 5.6 Risk Level (손절 기준가)

Countdown 13 완성 시 또는 Setup 9 완성 시 자동 계산되는 손절 기준:

```
매수 Risk = Countdown(또는 Setup) 구간 최저 True Low - 해당 봉의 True Range
매도 Risk = Countdown(또는 Setup) 구간 최고 True High + 해당 봉의 True Range
```

- 매수: 이 가격 아래로 저가가 도달하면 포지션 종료
- 매도: 이 가격 위로 고가가 도달하면 포지션 종료

### 5.7 Recycling (리사이클)

Countdown 진행 중 기존 카운트를 취소하고 새로 시작:

| 조건 | 설명 |
|---|---|
| Extended Setup (22 이상) | 같은 방향 Setup 카운트가 22봉 이상 연속 |
| Size Match | 새 Setup 크기가 이전 Setup의 100~200% 사이 |

차트에서 `R` 마커로 표시됨.

### 5.8 Cancellation (취소)

진행 중인 Countdown이 무효화되는 경우:

| 조건 | 설명 |
|---|---|
| 반대 방향 Setup 완성 | 매수 Countdown 중 매도 Setup 9 발생 |
| TDST 위반 | 매수 Countdown 중 어떤 봉의 True Low > TDST 저항선 |

차트에서 `✕` 마커로 표시됨.

## 6. 차트 화면 사용법

### 6.1 기본 조작

| 조작 | 방법 |
|---|---|
| 기간 이동 | 드래그 (좌우) |
| 확대/축소 | 마우스 휠 |
| Y축 자동 스케일 | 오른쪽 상단 **A** 버튼 |
| OHLC 확인 | 마우스를 캔들 위에 올림 → 좌측 상단에 표시 |

### 6.2 데이터 조회 방법

1. 심볼 입력창에 원하는 심볼 입력 (예: `BTCUSDT`, `ETHUSDT`)
2. 인터벌 선택: `15m`, `1h`, `4h`, `1d`, `3d`, `1w`
3. 기간 프리셋 클릭: `1M`, `3M`, `6M`, `1Y`, `2Y`
   또는 날짜 직접 입력
4. **Run** 버튼 클릭

최초 조회 시 Binance API에서 데이터 다운로드 (수초 소요). 이후 동일 기간 재조회는 Parquet 캐시에서 즉시 로드됨.

### 6.3 차트 오버레이 항목

| 표시 | 의미 |
|---|---|
| 캔들 아래 작은 숫자 (녹색) | 매수 Setup 카운트 1~8 |
| 캔들 위 작은 숫자 (빨강) | 매도 Setup 카운트 1~8 |
| 캔들 아래 `9` (녹색 원) | 매수 Setup 9 완성 |
| 캔들 위 `9` (빨강 원) | 매도 Setup 9 완성 |
| 캔들 아래 `9✓` | Perfected 매수 Setup 9 |
| 캔들 위 `9✓` | Perfected 매도 Setup 9 |
| 캔들 아래 숫자 (밝은 녹색) | 매수 Countdown 카운트 1~12 |
| 캔들 위 숫자 (밝은 빨강) | 매도 Countdown 카운트 1~12 |
| 캔들 아래 `13▲` (진녹색 화살표) | 매수 Countdown 13 완성 |
| 캔들 위 `13▲` (진빨강 화살표) | 매도 Countdown 13 완성 |
| 캔들 아래/위 `13+` | 13 보류(Deferral) 후 완성 |
| `R` (주황 원) | Recycling 발생 |
| `✕` (회색 원) | Cancellation 발생 |
| 녹색 수평 점선 | TDST 지지선 (S) |
| 빨강 수평 점선 | TDST 저항선 (R) |

### 6.4 시그널 클릭으로 이동

우측 패널의 시그널 목록에서 행을 클릭하면 해당 봉으로 차트가 자동 이동함.

## 7. 시그널 해석 및 매매 활용

### 7.1 Setup 9 — 추세 소진 경고

**매수 Setup 9** (`9` 녹색 원): 하락 추세가 9봉 지속 → 하락 에너지 소진 가능성

- **Perfected (`9✓`)**: 더 강한 소진 신호 — 저가가 더 낮게 형성됨
- **의미**: 저점 형성 가능 구간. 진입은 아직. Countdown 시작 대기.
- **주의**: Setup만으로 반전 확정 아님. 추세 지속 가능.

**매도 Setup 9** (`9` 빨강 원): 상승 추세 소진 → 고점 형성 가능

### 7.2 Countdown 13 — 진입 시그널

**매수 Countdown 13** (`13▲` 진녹색): 하락 소진이 구체화됨 → 매수(롱) 고려

- Setup 9 이후 13봉에 걸쳐 비연속적 확인
- `13+` (Deferral 후 완성): 더 강한 확인 — 시그널 품질 높음

**매도 Countdown 13** (`13▲` 진빨강): 상승 소진 → 매도(숏) 고려

### 7.3 TDST — 포지션 유지 기준

| 상황 | 활용 |
|---|---|
| 매수 진입 후 | TDST 지지선 (녹색 점선) 아래로 **종가 마감** 시 손절 또는 재검토 |
| 매도 진입 후 | TDST 저항선 (빨강 점선) 위로 **True Low > 저항선** 시 취소 |

**TDST 선 근처 가격**: 반등/반락 발생 빈도 높음 → 지지·저항으로 활용 가능.

### 7.4 Risk Level — 정량적 손절가

Countdown 13 완성 또는 Setup 9 완성 시 자동 계산됨. PnL 백테스터에서 이 값을 손절가로 사용.

```
매수 진입 시: 저가가 Risk Level 아래로 도달하면 포지션 종료
매도 진입 시: 고가가 Risk Level 위로 도달하면 포지션 종료
```

## 8. 통계 패널 사용법

차트 우측 패널에 시그널 이후 수익률 통계가 표시됨.

### 8.1 StatsCard (상단)

시그널 타입별 통계 카드:

| 항목 | 의미 |
|---|---|
| Count | 해당 타입 시그널 발생 횟수 |
| WR 5/10/20 | 5봉/10봉/20봉 후 수익 방향 비율 (%) |
| Avg 5/10/20 | 5봉/10봉/20봉 후 평균 수익률 |

카드를 클릭하면 수익률 분포 히스토그램이 펼쳐짐.

> **예시**: `buy_setup_9`의 WR20이 65%면 매수 Setup 9 발생 후 20봉 내 상승 확률 65%.

### 8.2 SignalList (하단)

| 컬럼 | 의미 |
|---|---|
| Time | 시그널 발생 시각 (한국 시간) |
| Type | 시그널 종류 |
| Dir | Buy(매수)/Sell(매도) |
| Entry | 진입가 (시그널 봉 종가) |
| +5/+10/+20 | 해당 봉 후 수익률 |

- **필터 버튼**: Setup/Countdown/Recycle/Cancel 별 필터링
- **CSV Export 버튼**: 시그널 목록 엑셀 저장

## 9. 스크리너 사용법

상단 **Screener** 탭 클릭.

### 9.1 기능

- 최대 20개 심볼 × 최대 6개 타임프레임 동시 스캔
- 각 심볼에서 최근 Setup/Countdown 카운트와 마지막 시그널 확인

### 9.2 사용법

1. **심볼 입력**: 쉼표 구분으로 입력 (예: `BTCUSDT,ETHUSDT,SOLUSDT`)
2. **인터벌 선택**: 체크박스로 원하는 타임프레임 선택
3. **Scan** 버튼 클릭 → 약 5~15초 소요 (데이터 수집 병렬 진행)

### 9.3 결과 해석

| 컬럼 | 의미 |
|---|---|
| Symbol / TF | 심볼과 타임프레임 |
| Close | 최근 종가 |
| Setup | 현재 Setup 카운트 (양수=매수, 음수=매도) |
| CD | 현재 Countdown 카운트 |
| Last Signal | 가장 최근 시그널 타입 |
| Signal Time | 시그널 발생 시각 |
| Perfected | ✓ = Perfected 시그널 |

### 9.4 스크리너에서 차트 연결

결과 행을 클릭하면 해당 심볼·타임프레임으로 자동 전환해 차트 조회 실행됨.

## 10. PnL 백테스터 사용법

상단 **PnL** 탭 클릭.

### 10.1 개요

- **진입**: Setup 9 완성 봉에서 진입 (매수 Setup 9 → 롱, 매도 Setup 9 → 숏)
- **손절**: Risk Level 돌파 (저가가 매수 Risk 아래, 또는 고가가 매도 Risk 위)
- **비교**: 종가 진입 vs 다음 봉 시가 진입 두 가지 결과 동시 산출

### 10.2 사용법

1. **Chart 탭** 또는 **PnL 탭**에서 심볼·인터벌·날짜 범위 설정
2. **PnL 탭** 클릭
3. **Run PnL** 버튼 클릭

### 10.3 결과 해석

**상단 통계 카드 (2개 비교)**

| 항목 | 의미 |
|---|---|
| 총 트레이드 | 전체 Setup 9 시그널 수 (미청산 포함) |
| 승/패 | 손절 발동 기준 수익/손실 건수 |
| 승률 | 손절 전 수익 달성 비율 |
| 평균 P&L | 청산 완료 트레이드 평균 수익률 |
| 평균 수익/손실 | 승리/패배 트레이드 각 평균 |
| 최대 수익/손실 | 단건 최대 결과 |
| 평균 보유 봉수 | 진입~손절까지 평균 봉 수 |

> **미청산**: 백테스트 기간 내 아직 Risk Level에 도달하지 않은 포지션. PnL 없음.

**하단 트레이드 목록**

| 컬럼 | 의미 |
|---|---|
| 시각 | 진입 시각 (✓ = Perfected) |
| 방향 | ▲Buy(롱) / ▼Sell(숏) |
| Risk | 손절 기준가 |
| 진입(Close) | 시그널 봉 종가 기준 진입가 |
| 진입(Next) | 다음 봉 시가 기준 진입가 |
| 청산가 | 손절 발동 가격 (= Risk Level) |
| 봉수 | 진입~청산까지 경과 봉 수 |
| 청산사유 | Stop(손절) / 미청산 |
| PnL(C) | 종가 기준 수익률 |
| PnL(O) | 다음봉 시가 기준 수익률 |

### 10.4 활용 방법

- **종가 vs 시가 비교**: 슬리피지(다음봉 시가)가 수익에 미치는 영향 확인
- **Perfected 시그널 필터**: `✓` 마크 있는 트레이드만 추려서 Perfected 효과 검증
- **타임프레임별 비교**: 4h, 1d에서 각각 실행해 최적 TF 탐색

## 11. 실시간 LIVE 기능

### 11.1 활성화 방법

1. **Chart 탭**에서 원하는 심볼·인터벌로 **Run** 실행
2. 데이터 로드 완료 후 툴바에 **LIVE** 버튼 등장 (회색 = 비활성)
3. **LIVE** 버튼 클릭 → 파란색으로 변경 (활성)

### 11.2 상태 표시

| 색상 | 의미 |
|---|---|
| 회색 | LIVE 비활성 — WS 연결 없음 |
| 노란색 (깜빡임) | 연결 중 |
| 파란색 (깜빡임) | 활성 — 실시간 가격 수신 중 |

활성 상태에서 현재 가격이 버튼 옆에 표시됨 (직전 종가 대비 초록/빨강).

### 11.3 LIVE 비활성화 (차트 탐색)

LIVE 활성 시 차트가 자동으로 현재 시점으로 스크롤됨. 과거 차트를 보고 싶으면:

→ **LIVE 버튼 클릭해 비활성화** → 차트 자유 탐색 가능

### 11.4 동작 설명

- 현재 열린 캔들이 실시간 업데이트됨 (tick 단위)
- 캔들 확정(close) 시:
  - 차트에 새 마커 추가 (새 Setup/Countdown 시그널 발생 시)
  - 텔레그램으로 알림 발송 (설정된 경우)

> **주의**: Binance 선물(USDT 마진) 가격과 현물 가격 간 최대 0.3% 차이 발생 가능. 이 대시보드의 실시간 데이터는 현물 WS 사용.

## 12. 텔레그램 알림 설정

Setup 9 또는 Countdown 13 시그널이 실시간으로 발생하면 텔레그램으로 알림을 받을 수 있음.

### 12.1 봇 생성

1. 텔레그램에서 `@BotFather` 검색
2. `/newbot` 명령 실행
3. 봇 이름·사용자명 입력
4. 발급된 **Bot Token** 복사 (형식: `123456789:ABCdef...`)

### 12.2 Chat ID 확인

1. 생성한 봇에게 아무 메시지 전송
2. 아래 URL 브라우저에서 열기 (토큰 교체):
   ```
   https://api.telegram.org/bot{TOKEN}/getUpdates
   ```
3. JSON에서 `"chat":{"id":12345678}` 부분의 숫자가 Chat ID

### 12.3 설정 파일 편집

`backend/.env`:

```
TELEGRAM_BOT_TOKEN=123456789:ABCdef...
TELEGRAM_CHAT_ID=12345678
```

백엔드 재시작 후 알림 활성화됨.

### 12.4 알림 내용

```
🟢 BUY SETUP 9
BTCUSDT 4h
Time: 26년1월15일 16:00
Price: $95,432.50
Perfected: ✓
Risk Level: $91,200.00
```

## 13. 매매 활용 시나리오

> **면책**: 이 도구는 교육·연구 목적. 실제 매매 결과를 보장하지 않음.

### 시나리오 A: Countdown 13 롱 진입 (기본)

```
1. 4h 차트에서 매수 Countdown 13 완성 확인 (13▲ 진녹색)
2. 시그널 봉의 다음 봉 시가에 롱 진입
3. Risk Level 가격을 손절가로 설정
4. TDST 저항선을 1차 목표가로 설정
5. 다음 매도 Setup 9 발생 시 청산 검토
```

### 시나리오 B: Perfected Setup 9 롱 진입 (단기 반등)

```
1. 4h 또는 1d에서 매수 Perfected Setup 9✓ 확인
2. 당일 또는 다음 봉 저점 매수 진입 고려
3. Risk Level 이하가 손절 기준
4. Countdown 카운트 진행 관찰 (13 도달 시 추가 보유 또는 익절)
```

### 시나리오 C: 멀티 타임프레임 확인

```
1. 스크리너에서 1d Setup 9 발생 심볼 탐색
2. 해당 심볼 4h 차트에서 Countdown 진행 확인
3. 4h에서도 Setup 9 또는 Countdown 진행 중이면 신뢰도 상승
4. 4h Countdown 13 완성 시 진입 타이밍
```

### 시나리오 D: TDST 레인지 트레이딩

```
1. 차트에서 TDST 지지선(녹색 점선)과 저항선(빨강 점선) 확인
2. 가격이 지지선 근처에서 반등 시 단기 롱
3. 가격이 저항선 근처에서 반락 시 단기 숏
4. 지지선 하향 이탈 종가 시 청산
```

### 시나리오 E: PnL 백테스터로 전략 검증

```
1. PnL 탭에서 원하는 심볼·타임프레임·기간 설정
2. Run PnL 실행
3. 승률·평균 PnL 확인
4. Perfected 시그널만 필터해서 일반 시그널 대비 성과 비교
5. 타임프레임별(1h vs 4h vs 1d) 결과 비교해 최적 TF 선택
```

### 시나리오 F: 실시간 알림 기반 진입

```
1. 텔레그램 알림 설정 완료
2. LIVE 버튼 활성화 상태로 유지
3. 알림 수신 시 차트 확인
4. 시그널 맥락(TDST 위치, 상위 TF 방향) 확인 후 진입 결정
```

## 부록: 자주 묻는 질문

**Q: 처음 실행 시 데이터 로딩이 오래 걸려요.**
A: 첫 조회 시 Binance API에서 전체 기간 데이터를 다운로드함. 1년치 1h = 약 3~10초. 이후 동일 기간은 Parquet 캐시에서 즉시 로드됨.

**Q: 특정 심볼/인터벌이 조회 안 돼요.**
A: Binance Futures에서 제공하는 심볼만 지원. `BTCUSDT`, `ETHUSDT`, `SOLUSDT` 등 USDT 마진 선물 심볼 사용.

**Q: Setup 카운트가 9에서 멈추고 10이 안 나와요.**
A: 정상. 9 완성 시 카운트 리셋되고 Countdown이 시작됨. 다음 Setup은 다시 1부터 시작.

**Q: LIVE 가격이 Binance 선물 차트와 조금 달라요.**
A: 현물 WS와 선물 가격의 0.1~0.3% 차이. 방향성 분석에는 영향 없음.

**Q: Countdown이 갑자기 사라져요.**
A: Cancellation 발생 (차트에 `✕` 표시). 반대 방향 Setup 9 완성 또는 TDST 위반으로 무효화됨.

**Q: 같은 봉에 Setup 9와 Countdown 13이 동시에 있어요.**
A: 가능. 이전 Setup의 Countdown 13 완성과 새 Setup 9가 같은 봉에서 발생한 경우.
